## Overview

PoolV3 is the core lending pool contract in Gearbox Protocol V3. It implements the ERC-4626 vault standard for deposits and withdrawals, manages borrowing by connected credit managers, and tracks interest rates and quota revenue.

**Contract Location:** `contracts/pool/PoolV3.sol`

## Key Features

- **ERC-4626 Compliance**: Full implementation of the vault standard for liquidity provision
- **ERC-20 Permit**: Supports gasless approvals via EIP-2612
- **Credit Manager Lending**: Lends funds to credit accounts through authorized credit managers
- **Dynamic Interest Rates**: Interest rates calculated based on pool utilization via interest rate model
- **Quota Revenue**: Additional revenue stream from quota interest on risky assets
- **Pausable**: Deposits, withdrawals, and transfers can be paused by authorized admins
- **Withdrawal Fees**: Optional fees on withdrawals sent to treasury

## Core Functions

### ERC-4626 Lending

#### deposit

```solidity
function deposit(uint256 assets, address receiver)
    external
    returns (uint256 shares)
```

Deposits underlying tokens into the pool in exchange for shares.

<ParamField path="assets" type="uint256">
  Amount of underlying tokens to deposit
</ParamField>

<ParamField path="receiver" type="address">
  Address to receive the minted shares
</ParamField>

**Returns:** Number of shares minted

**Requirements:**
- Pool must not be paused
- Receiver must not be zero address
- Assets and resulting shares must be non-zero

#### depositWithReferral

```solidity
function depositWithReferral(uint256 assets, address receiver, uint256 referralCode)
    external
    returns (uint256 shares)
```

Same as `deposit` but emits a referral event.

<ParamField path="referralCode" type="uint256">
  Referral code for tracking deposit source
</ParamField>

#### withdraw

```solidity
function withdraw(uint256 assets, address receiver, address owner)
    external
    returns (uint256 shares)
```

Burns shares to withdraw underlying tokens.

<ParamField path="assets" type="uint256">
  Amount of underlying to withdraw
</ParamField>

<ParamField path="receiver" type="address">
  Address to receive the underlying tokens
</ParamField>

<ParamField path="owner" type="address">
  Address to burn shares from (must approve if not msg.sender)
</ParamField>

**Returns:** Number of shares burned

#### redeem

```solidity
function redeem(uint256 shares, address receiver, address owner)
    external
    returns (uint256 assets)
```

Redeems shares for underlying tokens.

<ParamField path="shares" type="uint256">
  Number of shares to redeem
</ParamField>

**Returns:** Amount of underlying withdrawn

### Borrowing

#### lendCreditAccount

```solidity
function lendCreditAccount(uint256 borrowedAmount, address creditAccount)
    external
```

Lends funds to a credit account. Only callable by authorized credit managers.

<ParamField path="borrowedAmount" type="uint256">
  Amount to lend
</ParamField>

<ParamField path="creditAccount" type="address">
  Address of the credit account to receive funds
</ParamField>

**Requirements:**
- Caller must be an authorized credit manager
- Borrowed amount must not exceed credit manager's debt limit
- Total borrowed must not exceed pool's total debt limit
- Sufficient available liquidity must exist

```solidity
// Example: Credit manager borrows 1000 USDC for a credit account
pool.lendCreditAccount(1000e6, creditAccountAddress);
```

#### repayCreditAccount

```solidity
function repayCreditAccount(uint256 repaidAmount, uint256 profit, uint256 loss)
    external
```

Updates pool state after debt repayment. Only callable by credit managers.

<ParamField path="repaidAmount" type="uint256">
  Amount of principal repaid
</ParamField>

<ParamField path="profit" type="uint256">
  Pool's profit after repayment (mints shares to treasury)
</ParamField>

<ParamField path="loss" type="uint256">
  Pool's loss after repayment (burns treasury shares)
</ParamField>

<Note>
The credit manager must transfer underlying from the credit account to the pool before calling this function.
</Note>

- Base interest proportional to borrowed amount
- Quota revenue



Returns the amount available to borrow for a specific credit manager, considering:
- Credit manager's debt limit
- Total pool debt limit  
- Interest rate model constraints
- Available liquidity

## Example Usage

### Depositing Liquidity

```solidity
// Approve pool to spend underlying
IERC20(underlying).approve(address(pool), 10000e6);

// Deposit 10,000 USDC
uint256 shares = pool.deposit(10000e6, msg.sender);

// Or deposit with referral
uint256 shares = pool.depositWithReferral(10000e6, msg.sender, 12345);
```

### Withdrawing Liquidity

```solidity
// Withdraw 5,000 USDC
uint256 sharesBurned = pool.withdraw(5000e6, msg.sender, msg.sender);

// Or redeem specific number of shares
uint256 assetsReceived = pool.redeem(shares / 2, msg.sender, msg.sender);
```

### Checking Pool State

```solidity
// Check available liquidity
uint256 available = pool.availableLiquidity();

// Check expected liquidity (includes borrowed + interest)
uint256 expected = pool.expectedLiquidity();

// Check current rates
uint256 borrowRate = pool.baseInterestRate(); // Rate borrowers pay
uint256 lenderRate = pool.supplyRate(); // Rate lenders receive

// Check how much a credit manager can borrow
uint256 borrowable = pool.creditManagerBorrowable(creditManagerAddress);
```

## Security Considerations

<Info>
**First Depositor Protection**: Small amount of shares should be minted to a dead address before allowing borrowing to prevent first depositor front-running attacks.
</Info>

<Info>
**Reentrancy Protection**: All state-changing functions use the `nonReentrant` modifier.
</Info>

<Info>
**Pausability**: Emergency pause only blocks deposits/withdrawals/transfers, not borrowing/repayment, to allow liquidations to proceed.
</Info>
